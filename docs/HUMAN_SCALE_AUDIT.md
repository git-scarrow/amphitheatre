# Human-scale references — placements, metadata, scale assumptions, validation

**Purpose.** Make the Scenario E civic bowl read at human scale — in the web
viewer and on the boards — with *calibrated schematic* references, never
decorative entourage. Every visible figure and dimension is generated from one
source layer; nothing is hand-drawn; the audit gate fails the package if any
presentation output drifts from the source.

Canon preserved: open-air landscape venue (no walls, ceiling, tall upstage
shell, or fly tower implied by any ref); stage refs surface DESIGN_CANON
Rule 9 OPEN; no ref adds silhouette inside the measured az-330 bay-view
corridor.

## Source layer

`vectors_geojson/human_scale_refs.geojson` — generated by
`scripts/build_human_scale_refs.py` from governing geometry only
(EPSG:6494 intl ft, NAVD88 (Geoid12A) intl ft, deterministic):

- `vectors_geojson/terrace_treads.geojson` (Scenario E emitted treads)
- `vectors_geojson/bowl_zones.geojson` (stage, cross-aisle, promenade,
  ADA ramps/landings, treatment cell)
- `analysis/in_situ_normalization/stage_typology_scores.json` (P_opt
  provisional deck — pocket pinches)
- `design_open_low/stage_floor.geojson` (stage-front focal point lineage)
- `analysis/tier_emission/ambitious_shaped_bowl_seating/geometry.geojson`
  (promoted row 20, ambitious option)
- `vectors_geojson/site_context.geojson` (open lawn, bay-view corridor)
- `dem/proposed_grade_1ft.tif` (sampled ground where no design surface
  declares an elevation)

## Scale assumptions

- Figure **heights are exact in data units** (ft); silhouettes are schematic.
- Section panels that carry figures use equal-aspect axes (true 1:1).
- Board renderers record each drawn figure's **vertex-derived** vertical
  extent in `boards/board_sources.json` / `analysis/decision_packet/
  sources.json` (`human_scale.figures[].drawn_height_ft`) — measured from the
  actual polygon, not echoed from the input.
- Plan markers claim position only, not height.

## Validation (scripts/audit_in_situ_package.py · check_human_scale_refs)

Hard failures, each verified by sabotage tests on 2026-06-12:

1. **Metadata** — missing required fields, a standing height outside
   {5.0, 5.75, 6.25}, a seated eye ≠ 3.94, a wheelchair eye ≠ 3.90, or
   `blocks_bay_view` ≠ false; any required placement absent.
2. **Extents** — any ref outside the treads/zones/lawn bounds (+25 ft).
3. **Float/sink** — `declared:` grounds re-derived from their layer
   (±0.01 ft) and checked against the proposed-grade raster (±0.35 ft,
   structure-deck refs exempted); `raster:` grounds re-sampled (±0.35 ft).
4. **Dimension honesty** — geometric length vs `length_ft` (±0.15 ft);
   the 50-ft bar exact.
5. **Bay view** — the stage's own visual-envelope rule applied to refs:
   inside the measured bay-view corridor no head may rise above the
   camera→NW-rim-silhouette (618.5) grazing ray from the required mid-row
   viewpoint.
6. **Presentation integrity** — viewer payload or board manifests carrying
   human figures without the source layer ("presentation-only humans");
   viewer heights/grounds not byte-equal to the source; any drawn figure
   without a source ref; `drawn_height_ft` vs `height_ft` beyond 0.05 ft;
   no wheelchair figure reaching the viewer or the ADA-relevant boards.
